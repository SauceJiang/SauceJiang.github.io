我的第一篇代码笔记

从原始数组到干净表格：一个可复用的数据清洗流水线

在机器学习项目里，数据清洗往往是第一步，也是最耗时的环节。本文以 DLKcat 酶动力学数据的清洗脚本为例，拆解一个结构清晰、可复用的 Python 数据处理模块，分享从 .npy 原始数组到规整 CSV 的全过程。

1. 背景与目标

我们手里有一份原始的酶催化反应数据，每个样本包含三个部分：

  • proteins.npy：蛋白质的 token ID 序列（不定长列表）
  • compounds.npy：化合物的 token ID 序列（不定长列表）
  • regression.npy：对应的目标值 log10(kcat)

这些数据以 NumPy 对象数组的形式存储，不方便直接用于建模。我们的目标是把它清洗成一张干净的 CSV 表格，并在这个过程中：

  • 剔除无效样本（空序列、缺失目标值、极端异常值）
  • 去除重复数据
  • 将不定长序列编码成可以存入 CSV 的字符串格式
  • 保证整个流程可复现、可追溯

3.1路径与目录约定

  • 用Path(__file__).resolve().parents[1]定位到项目根目录，保证脚本在任何位置执行都能找到正确的数据。
  • 输入、输出目录分离，原始数据只读不写，清洗结果写入interim/cleaned/，符合数据科学项目的标准分层
